The Dual-Core Engine

Logic Core (WASM)

Powered by Rapier2D, calculated in a background simulation. Handles rigid-body dynamics, precise collisions, and momentum vectors at 60Hz.

Sync-Layer

The "Vibe Proxy" bridges the two cores, mapping physical translations to React transforms with sub-pixel precision.

Next-Gen Physics Layout

VibeDev Pro Layout goes beyond standard FLIP transitions. It introduces Physical Impulse FLIP.

The "Impulse" FLIP

Standard animations use simple mathematical springs. VibeDev converts the layout delta into a physical impulse. If elements move, they have weight. If they collide, they displace each other.

<vibe.div 
  layout="physics" 
  layoutId="header"
  restitution={0.8}
>
  Physical Header
</vibe.div>

Shared Momentum

Using layoutId, physical states are transferred across components. An unmounting component passes its velocity to the new mounting instance.

Auto Scale Correction

Powered by CSS Variables (--vibe-counter-scale). Children automatically counteract parent scale distortion without boilerplate props.

Layout Animation: API Reference

1. The layout Prop

Controls how the element reacts to layout changes. Supported values:

  • true: Standard FLIP animation using high-damping springs.
  • "position": Animates changes in position only, snapping size instantly (perfect for text/images).
  • "physics": Converts the transition into a physical impulse in the Rapier world.
<vibe.div layout="physics">...</vibe.div>

2. layoutId (Shared Transitions)

Connects two elements across different component trees. When element A unmounts and element B mounts with the same ID, B will physically "grow" out of A.

{selectedID === item.id && (
  <vibe.div layoutId="highlight-box" className="outline" />
)}

3. LayoutGroup

Synchronizes layout shifts across independent elements. If one element's layout changes and affects its neighbors, wrapping them in LayoutGroup ensures they all animate together physically.

import { LayoutGroup } from 'vibedev-motion';

<LayoutGroup>
  <Accordion />
  <Accordion />
</LayoutGroup>

4. Technical Source (Impulse-FLIP)

How we solve layout deltas using WASM Physics:

// Internal Hook Snippet
const invertX = firstRect.left - lastRect.left;
const invertY = firstRect.top - lastRect.top;

if (layout === 'physics' && body) {
  // Convert pixels to physical impulse units
  const impulse = { x: invertX * 0.1, y: invertY * 0.1 };
  body.applyImpulse(impulse, true);
}
